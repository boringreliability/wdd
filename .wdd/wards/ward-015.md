# Ward 015: Manual Smoke Test Protocol

## Scope
Add mandatory `## Manual Smoke Test` section to the ward template body and teach `wdd complete` to extract and replay that section in completion output. This makes it structurally impossible to complete a Ward without having documented how a human can verify the work manually. Additionally, update the Claude/Cursor adapter content to instruct AI: spin up the solution and guide the human through the smoke test before claiming Gold-phase approval.

This Ward addresses the third forcing function of WDD methodology — automated tests prove functions work, integration tests prove they're wired, **manual smoke tests prove a human can use them**.

## Inputs
- Ward template body in [src/commands/ward-create.ts](src/commands/ward-create.ts) — embedded template
- Ward template file `.wdd/templates/ward.md` — generated by [src/commands/init.ts](src/commands/init.ts)
- `completeWard()` from Ward 5 (with reminders from Ward 9)
- Frontmatter parser from Ward 1
- Adapter content in [src/templates/adapter-content.ts](src/templates/adapter-content.ts)

## Outputs
- New `extractSection(body, heading)` utility in `src/utils/section.ts` — extracts markdown section by `## Heading`
- Updated ward template body (both embedded and file template) with `## Manual Smoke Test` section
- Updated `completeWard()` that prints smoke test recap from ward body
- Updated `getSharedContent()` adapter text instructing AI on smoke test protocol

## Specification

### Ward Template Addition
New obligatory section between `## Must DO` and `## Verification`:

```markdown
## Manual Smoke Test
### Setup
{Exact commands to spin up — npm link, dev server, build, etc.}

### Steps
1. Run: `{exact command}`
   Expected: `{exact output or behavior}`
2. Run: `{next command}`
   Verify: `{what to look for}`

### Pass criteria
- [ ] {concrete observable thing}
- [ ] {concrete observable thing}
```

### Section Extraction Utility
```typescript
export function extractSection(body: string, heading: string): string
```
- Takes markdown body and a heading name (e.g., `"Manual Smoke Test"`)
- Returns content between `## {heading}` and the next `## ` heading (or EOF)
- Returns empty string if heading not found
- Trims leading/trailing whitespace
- Lives in `src/utils/section.ts` for reuse in future Wards (e.g., Ward 16 might extract `## Tests` for export inventory)

### `wdd complete` Enhancement
After the existing reminder steps (commit, CONTEXT.md), if the ward body contains a `## Manual Smoke Test` section, append step:

```
📋 Manual Smoke Test (replay before declaring complete next time):
{extracted smoke test content, indented}
```

If no smoke test section exists (older wards 1-14), skip silently — backwards compatible.

### Adapter Content Update
Append new section to `getSharedContent()` in adapter-content.ts:

> ## Manual Smoke Test Protocol (MANDATORY before Gold approval)
>
> Before requesting Gold-phase approval from the human, you MUST:
> 1. Read the `## Manual Smoke Test` section from the active ward file
> 2. Spin up the solution as the Setup section describes (build, npm link, dev server, etc.)
> 3. Provide the exact Steps as copy-pasteable commands to the human
> 4. Wait for the human to run them and report results
> 5. Only after they confirm "smoke test passed" may you ask for Gold approval
>
> Do not claim Gold complete based on automated tests alone. The human must have *used* the thing.
