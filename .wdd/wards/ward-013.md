# Ward 013: Skill Commands

## Scope
Add user-invocable skill commands (`/wdd`, `/ward`, `/ward-new`) to the Claude Code skill generated by `wdd bootstrap claude`. These are AI workflow instructions — not CLI wrappers — that tell the AI what to do in each WDD phase. The skill commands orchestrate the AI's behavior and enforce checkpoint discipline.

## Inputs
- `getClaudeSkill()` from Ward 11
- Claude Code skill format: `user_invocable_skills` section with command triggers

## Outputs
- Updated `getClaudeSkill()` with three skill commands embedded
- Updated `getSkillContent()` with command reference section
- Tests verifying command content in generated skill file

## Specification

### Skill Commands

#### `/wdd` — Session Context
Instruction to AI:
1. Run `wdd session` and read the output
2. Run `wdd validate` to check project health
3. Summarize for the user: current state, active ward, any blockers
4. Ask what the user wants to work on

#### `/ward` — Continue Current Ward
Instruction to AI:
1. Run `wdd session` to find current ward and its status
2. Based on status:
   - **planned**: Transition to red (`wdd ward status N red`). Write tests per ward spec. Present tests. STOP — wait for approval.
   - **red**: Present tests if not yet shown. STOP — wait for approval.
   - **approved**: Transition to gold (`wdd ward status N gold`). Implement until tests pass. Present implementation. STOP — wait for approval.
   - **gold**: Run tests. If passing, present for approval. STOP — wait for approval.
   - **complete**: Tell user ward is done. Suggest `/ward-new` or next ward.
   - **blocked**: Tell user what's blocking and suggest resolution.
3. NEVER skip a checkpoint. NEVER proceed without explicit human approval.

#### `/ward-new` — Create New Ward
Instruction to AI:
1. Ask user for: ward name, epic (suggest existing epics from `.wdd/epics/`)
2. Run `wdd ward create "Name" --epic <slug>`
3. Open the created ward file and write the full spec: Scope, Inputs, Outputs, Specification, Tests table, Must NOT, Must DO, Verification
4. Transition to red: `wdd ward status N red`
5. Write the test file per the Tests table
6. Present tests to user. STOP — wait for approval.

### Claude Code Skill Format
The commands are embedded in the skill markdown as sections that Claude Code recognizes as invocable:

```markdown
## User-Invocable Skills

### /wdd
{instruction content}

### /ward
{instruction content}

### /ward-new
{instruction content}
```
